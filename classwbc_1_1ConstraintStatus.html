◆ activation

double wbc::ConstraintStatus::activation

Constraint activation. Has to be between 0 and 1. Will be multiplied with the constraint weights. Can be used to (smoothly) switch on/off the constraints.

◆ config

Last time the constraint reference values was updated.

◆ timeout

int wbc::ConstraintStatus::timeout

Can be 0 or 1. Will be multiplied with the constraint weights. If no new reference values arrives for more than config.timeout time, this value will be set to zero.
